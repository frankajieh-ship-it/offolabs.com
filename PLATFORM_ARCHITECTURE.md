## ğŸ¯ Key Architectural Decisions

### 1. Monorepo Structure
- **Decision:** Separate `server/` and `frontend/` directories
- **Rationale:** Independent deployment, clear separation of concerns
- **Trade-off:** Some code duplication (types), but better maintainability

### 2. REST API over GraphQL
- **Decision:** RESTful API design
- **Rationale:** Simpler, more familiar, easier to cache
- **Trade-off:** More endpoints, but better clarity

### 3. Socket.IO for Real-Time
- **Decision:** WebSocket via Socket.IO
- **Rationale:** Bidirectional communication, fallback support
- **Trade-off:** More complex than polling, but much better UX

### 4. MongoDB (Document DB)
- **Decision:** MongoDB over PostgreSQL
- **Rationale:** Flexible schema, good for rapid development
- **Trade-off:** Less strict relationships, but more agile

### 5. Next.js App Router
- **Decision:** App Router over Pages Router
- **Rationale:** Modern React features (Server Components)
- **Trade-off:** Newer API, but future-proof
