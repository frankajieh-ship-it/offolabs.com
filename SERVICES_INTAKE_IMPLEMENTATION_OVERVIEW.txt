================================================================================
  VALIDATION RULES
================================================================================

Client-Side Validation (Real-time)
────────────────────────────────────
Name:           Non-empty string
Email:          Non-empty + regex pattern match
Company:        Optional (no validation)
Service Type:   Must be selected
Budget:         Optional (no validation)
Timeline:       Optional (no validation)
Description:    Non-empty + minimum 20 characters

Server-Side Validation (Security)
────────────────────────────────────
Name:           Required, non-empty
Email:          Required, valid format (regex)
Service Type:   Required, non-empty
Description:    Required, minimum 20 characters

Error Codes:
────────────────────────────────────
INVALID_NAME                 — Name is required
INVALID_EMAIL                — Email is required
INVALID_EMAIL_FORMAT         — Invalid email format
INVALID_SERVICE_TYPE         — Service type is required
INVALID_DESCRIPTION          — Description is required
DESCRIPTION_TOO_SHORT        — Min 20 characters
INTERNAL_ERROR               — Server error
